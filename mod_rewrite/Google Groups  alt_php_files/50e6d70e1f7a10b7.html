Hi Justin,

Maybe I got it wrong, but aren't your tests resulting in exactly what you
wanted?

You pattern reads (the last bit is strange though):
From the beginning of the string 1 or more alphanumeric characters:
^([[:alnum:]]+)
PLUS
a forward slash: /
PLUS
1 or more alphanumeric character: ([[:alnum:]]+)
PLUS
NO single character: [^.]

The only thing strange in your pattern is the [^.] at the end, maybe you've
meant to exclude a single dot? Then it should read: [^\.]

Hope this helps,

Yes, the tests verify the regex that I _was_ using was good. The problem
was that even though the regex was OK, the URIs weren't being re-written
as expected. Things have changed, but here is the scenario:

A client (let's say Company1) has a web site that includes portfolios of
other clients (Company2). The way the site was first programmed, each of
the other companies had (and still do) a directory [/Company2.com] to
keep their pages in. They also had a separate directory named
'portfolio' that was inside their client directory.

Each portfolio has sections, pages, and items. Each needs to be accessed
via URIs like:

/Company1/section/
/Company1/section/page/
/Company1/section/page/id/

each should go to the respective page:

/Company1/portfolio.php?section=section
/Company1/portfolio.php?section=section&page=page
/Company1/portfolio.php?section=section&page=page&id=id

I now have the following .htacces that was to be put in each of the
client folders:

RewriteEngine On
Options +FollowSymlinks
RewriteBase /client_name
RewriteRule ^(.*)/portfolio/(.*)/(.*)/(.*)/$
$1/port-item.php?section=$2&page=$3&id=$4
RewriteRule ^(.*)/portfolio/(.*)/(.*)/$ $1/port-page.php?section=$2&page=$3
RewriteRule ^(.*)/portfolio/(.*)/$ $1/portfolio.php?section=$2
RewriteRule ^(.*)/portfolio/$ $1/portfolio.php

Unfortunately, it didn't work as expected. It _seems_ that all the
requests are thrown to /client_name/portfolio.php, but never with any
variables passed via the query string.

An example link from the site is:
<a href="/Company1/portfolio/a/"> which should translate to:
/Company1/portfolio.php?section=a but instead seems to be:
/Company1/portfolio.php

Any help / insight is greatly appreciated!

Hi Justin,

try and append the "no escape" [NE] flag to the rules, example:

RewriteRule ^(.*)/portfolio/(.*)/$ $1/portfolio.php?section=$2 [NE]

I think Apache tries to escape "?", "&" and "=" with the equivalent hexcodes
"%3F", "%26" and "%3D".

Hope this helps,
JOn

PROBLEM SOLVED! It all came to be that you cant call your non-existant
directory the same as your script. IOW: I cahnged portfolio.php to
port.php and now calling /cleient_dir/portfolio/ works as expected!

I DID NOT SEE THIS IN DOCUMENTATION ANYWHERE!!!

So now, .htaccess:
RewriteEngine On
Options +FollowSymlinks
RewriteBase /client_dir

RewriteRule portfolio/(.*)/(.*)/(.*)/$  port.php?var1=$1&var2=$2&var3=$3
[NC]
RewriteRule portfolio/(.*)/(.*)/$  port.php?var1=$1&var2=$2
RewriteRule portfolio/(.*)/$  port.php?var1=$1
RewriteRule portfolio/ port.php
