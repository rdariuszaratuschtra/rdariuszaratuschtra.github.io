Hi there,

I'm running Linux with Apache 1.3.27

URL is:
/directory/subdirectory/hello.html?Session=4711

I have the following rule:
RewriteRule ([^/]+)/([^/]+)/([^/]+)\.html(.*) index.php$4&variable=$1

and want to get:
index.php?Session=4711&variable=directory

but what I do get is:
index.php&variable=directory

which results in "not found"

why does Apache somehow not get $4
If I leave out $1, then $4 is there (using the same URL):

RewriteRule ([^/]+)/([^/]+)/([^/]+)\.html(.*) index.php$4

works fine.

Any answers?

> URL is:
> /directory/subdirectory/hello.html?Session=4711

> I have the following rule:
> RewriteRule ([^/]+)/([^/]+)/([^/]+)\.html(.*) index.php$4&variable=$1

> and want to get:

> but what I do get is:
> index.php&variable=directory

> which results in "not found"

> RewriteRule ([^/]+)/([^/]+)/([^/]+)\.html(.*) index.php$4

this is pure luck. actually $4 is empty in both cases, only with the
2nd example you don't clobber the query string by using ? or & in the
target expression.

the query string is _not_ usable in a RewriteRule source regex! either
do it the canonical way:

RewriteCond %{QUERY_STRING} (.*)    
# this can be any regex in case you need
RewriteRule ([^/]+)/([^/]+)/([^/]+)\.html index.php?%1&variable=$1  
# %1 means first match in RewriteCond

or in your case (whole query string needed and not only parts) the
more elegant:

RewriteRule ([^/]+)/([^/]+)/([^/]+)\.html
index.php?%{QUERY_STRING}&variable=$1

or even:

RewriteRule ([^/]+)/([^/]+)/([^/]+)\.html index.php?variable=$1 [QSA]

QSA stands for query string attach and means not to clobber the old
query string but to attach the new part to whatever was there...
